Pointers & References

🟡 Basic · Lesson 8 ⏱ ~25 min read

What is a Pointer?

A pointer is a variable that stores the memory address of another variable. Instead of holding a data value directly, it holds the location in memory where a value lives.

Every variable in C++ occupies memory. A pointer lets you directly access and manipulate that memory — which is why C++ is so powerful for systems programming.

cpp 
int age = 25;        // normal variable — stores value 25
int* ptr = &age;     // pointer — stores the ADDRESS of 'age'

// & means "address of"
// * means "pointer to" (in declaration)

cout << age;    // Output: 25       (the value)
cout << &age;   // Output: 0x7fff5 (the address — will differ)
cout << ptr;    // Output: 0x7fff5 (same address, stored in ptr)
cout << *ptr;   // Output: 25      (dereference — value AT the address)
Key insight: The * symbol has two different meanings in C++: In a declaration (int* ptr) it means "this is a pointer." Used on an existing pointer (*ptr) it means "give me the value at this address" — this is called dereferencing.

Pointer Syntax

Let's break down every piece of pointer syntax clearly:

cpp 
// DECLARING a pointer
int* ptr;          // ptr can hold the address of an int
double* dPtr;      // dPtr can hold the address of a double
char* cPtr;        // cPtr can hold the address of a char

// INITIALIZING a pointer
int num = 42;
int* ptr = #   // &num gives the address of 'num'

// DEREFERENCING — reading the value
cout << *ptr;      // Prints: 42

// DEREFERENCING — modifying the value
*ptr = 100;
cout << num;       // Prints: 100 (original variable changed!)

// NULL pointer (safe initialization)
int* nullPtr = nullptr;  // Modern C++ — use nullptr, not NULL

Why Use Pointers?

  • Pass large data efficiently — pass a pointer instead of copying the whole object
  • Dynamic memory allocation — allocate memory at runtime with new
  • Data structures — linked lists, trees, and graphs are built with pointers
  • Function out-parameters — return multiple values from a function

Pointer Arithmetic

Pointers understand the size of the type they point to. Adding 1 to an int* moves the address forward by 4 bytes (size of int), not 1 byte.

cpp 
int arr[] = {10, 20, 30, 40};
int* p = arr;        // Points to first element (arr[0])

cout << *p;          // 10
cout << *(p + 1);    // 20 — moves 4 bytes forward
cout << *(p + 2);    // 30
cout << *(p + 3);    // 40

// Incrementing the pointer
p++;                  // now points to arr[1]
cout << *p;          // 20

References — A Safer Alternative

A reference is an alias for an existing variable. Unlike a pointer, a reference:

  • Must be initialized when declared
  • Cannot be reassigned to refer to a different variable
  • Cannot be null
  • Has cleaner syntax — no need to dereference with *
cpp 
int x = 10;
int& ref = x;    // ref is an alias for x

cout << ref;     // 10 — same as cout << x
ref = 50;        // modifying ref modifies x
cout << x;       // 50

// Common use: pass by reference in functions
void doubleIt(int& val) {
    val *= 2;     // modifies the original variable
}

int n = 7;
doubleIt(n);
cout << n;       // 14

Pointers vs References — When to Use Which

cpp 
// Use REFERENCE when:
// - The parameter should not be null
// - You won't reassign to point elsewhere
void increment(int& val) { val++; }

// Use POINTER when:
// - The value might be nullptr (optional parameter)
// - You need to reassign to point to different memory
// - Working with arrays or dynamic memory
void process(int* arr, int size) {
    for (int i = 0; i < size; i++)
        arr[i] *= 2;
}
⚠️ Dangling Pointers: A dangling pointer points to memory that has been freed or gone out of scope. Always set pointers to nullptr after deleting them, and use smart pointers (covered in Advanced) to avoid this entirely.
